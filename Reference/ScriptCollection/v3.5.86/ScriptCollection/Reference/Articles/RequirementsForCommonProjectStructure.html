Required tools for CommonProjectStructure

Hints

ScriptCollection contains scripts to implement a repository-structure defined by the CommonProjectStructure. To be able to run the scripts specified by the CommonProjectStructure successfully some tools may be required dependent on your project. For typically used tools the Tools-section contains a short description how to install them.

Requirements

Configuration

localhost.private-hosts-entry

*.localtest.me is a not-existing domain which can be used to host a HTTPS-server with a trusted certificate on the local development machine. For this reason you have to add an entry like 127.0.0.1 *.localtest.me to the hosts-file on your local development-machine. After this the scripts inside of the code-unit can generate a certificate for the domain codeunitname*.localtest.me which can be used for local development-purposes. See the ReadMe for more information.

Tools

ar

ar is an archive-program. It can be installed with apt-get install binutils on a Debian- or Ubuntu-system. It can be installed using MinGW on a Windows-system.

coverage

coverage is required to create test-coverage-reports for python-projects. The package can be installed using pip install coverage.

docfx

DocFX is a tool to generate HTML-references by markdown-files. It is primary used for references based on documentation-comments in the source-code of .NET-projects. The commandline-tool can be installed using choco install docfx.

docker

docker is required to create docker-images from a dockerfile. Install-instructions can be found here.

docker pushrm

pushrm is a commandline-tool to edit descriptions of a docker-image on hub.docker.com. Install-instructions can be found here.

dotnet

dotnet is the CLI-command to provide commands for working with .NET projects and run .NET applications. Install-instructions for the commandline-tool can be found here.

dotnet cyclonedx

CycloneDX for .NET-projects is a tool to generate sbom.xml in the BOM-artifact for .NET-projects. The commandline-tool can be installed using dotnet tool install --global CycloneDX.

gh

gh is a GitHub's commandline-tool to interact with github.com from commandline. The commandline-tool can be installed using choco install gh. Further installation-instructions are on the official website.

git

git is a common version control system.

gitversion

gitversion is a tool to calculate the project-version based on the git-history. The commandline-tool can be installed using choco install gitversion.portable in a shell with elevated privileges.

java

java is the commandline-tool for running Java-programs.

ng

ng is a CLI-tool to manage angular-projects. The commandline-tool can be installed using npm i @angular/cli.

npm

npm is the package-manager of node. The command npm is usually be available after you have installed node.

nuget

NuGet is required to create NuGet-packages. The commandline-tool can be installed using choco install nuget.commandline.

python

Python is a common script-language.

reportgenerator

ReportGenerator is required to generate HTML-reports from test-coverage-reports. The commandline-tool can be installed using choco install reportgenerator.portable.

scriptcollection

ScriptCollection is required to run the scripts (Build.py, etc.). The package can be installed using pip install ScriptCollection.

sh

The shell sh is already pre-installed on most linux-systems. On Windows a windows-port called sh.exe is contained in git for windows and can be found usually in C:\Program Files\Git\usr\bin.

sn

The Strong Name tool (Sn.exe) helps sign assemblies with strong names. It is contained the Windows SDK.

swagger

Swagger is required to generate the APISpecification-artifact for .NET-WebAPI-Projects. The commandline-tool can be installed using dotnet tool install --global swashbuckle.aspnetcore.cli.

t4

t4 is a tool to generate C#-files from T4-templates. It can be installed using dotnet tool install --global dotnet-t4. It is an alternative implementation for TextTransform.

tar

tar is an archive-program. It can be installed with apt-get install tar on a Debian- or Ubuntu-system. It can be installed using MinGW on a Windows-system.

twine

Twine is a utility for publishing python packages to PyPI and other repositories. The commandline-tool can be installed using pip install twine.
